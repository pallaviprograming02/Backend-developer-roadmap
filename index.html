📝 Sample Express.js Code

server.js 
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

// Middleware
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// Routes
app.get('/api/users', (req, res) => {
    const users = [
        { id: 1, name: 'John', email: 'john@example.com' },
        { id: 2, name: 'Jane', email: 'jane@example.com' }
    ];
    res.json({ success: true, data: users });
});

app.post('/api/users', (req, res) => {
    const { name, email } = req.body;
    // Validate input
    if (!name || !email) {
        return res.status(400).json({ 
            success: false, 
            message: 'Name and email required' 
        });
    }
    res.status(201).json({ 
        success: true, 
        message: 'User created',
        data: { id: 3, name, email }
    });
});

// Error handling middleware
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({ 
        success: false, 
        message: 'Something went wrong!' 
    });
});

app.listen(PORT, () => {
    console.log(`🚀 Server running on port ${PORT}`);
});

Ace your backend developer interviews

Common Questions

What is the difference between PUT and PATCH?

PUT replaces the entire resource, while PATCH updates only the specific fields. PUT is idempotent (same result if called multiple times), while PATCH may not be.

Explain RESTful API principles

REST APIs should be: Stateless (no client context on server), use HTTP methods properly (GET, POST, PUT, DELETE), have resource-based URLs, support caching, and return JSON/XML data.

What is the event loop in Node.js?

The event loop allows Node.js to perform non-blocking I/O operations by offloading operations to the system kernel. It handles callbacks, timers, and async operations in a single thread.

Scenario-Based Questions

How would you handle 10,000 concurrent connections?

Use clustering to utilize all CPU cores, implement load balancing, use caching (Redis), optimize database queries, consider microservices, and use async/await for non-blocking operations.

How do you secure a REST API?

Use HTTPS, implement JWT authentication, validate all inputs, use rate limiting, implement CORS properly, sanitize data to prevent XSS, use parameterized queries for SQL injection prevention.

What is your approach to debugging production issues?

Check logs first, use monitoring tools, reproduce the issue in staging, use debugging tools, add detailed logging, use APM tools, check resource usage, and review recent changes.

Database Design Questions

When would you use SQL over NoSQL?

Use SQL when you need ACID compliance, complex queries with joins, structured data, strict schema, and transactions. NoSQL is better for flexible schemas, horizontal scaling, and simple queries.

How do you optimize slow database queries?

Add appropriate indexes, optimize JOINs, avoid SELECT *, use query caching, analyze with EXPLAIN, limit result sets, denormalize when needed, and partition large tables.
